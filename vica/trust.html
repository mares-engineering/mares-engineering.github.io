Private-alpha trust and install safety

Trust, Privacy, Install Safety

No hype. No surprises.

VICA is a private-alpha, install-sensitive race-awareness instrument. This page explains what can be expected today and where control should come from: explicit enabling, explicit disable, explicit support boundaries.

Install posture

Controlled delivery, controlled expectations

VICA does not have a public installer in this phase. Alpha access and packaging are controlled and documented conservatively so users can verify what is run on their machine.

  • Install behavior is private-alpha testing only, not a public product launch.
  • No payment flow, account system, or licensing portal is linked from this site.
  • Do not assume broad simulator support; public pages only describe currently documented behavior.
  • Keep only one validated test path active per machine while evaluating.

Disable and uninstall expectations

Fail-open behavior should be easy to reach.

The driver should always be able to stop or hide the display path quickly. If any cue affects vision, attention, or performance, stop and disable before continuing.

  1. Use the documented disable path before driving if VICA is not needed.
  2. For immediate pauses, exit driving and disable VICA for the next launch.
  3. Remove or disable startup hooks that inject VICA into a simulator session.
  4. Delete or archive local package folders only after stopping associated runtime layers and services.

Data boundaries

Privacy is local-first by default.

VICA public guidance does not claim cloud-first operation. The HUD behavior is designed so race-facing output can function without external account infrastructure.

  • Current public docs do not describe server-side storage of session data.
  • Any logs or temporary files are only generated if you explicitly run tooling that enables them.
  • Do not share private session traces beyond private alpha support needs.
  • There are no public statements of certification, official approval, or zero-risk operation.

Trust posture

Reliability through restraint.

Missing, stale, invalid, or low-confidence inputs should be treated as an absence of confidence. VICA should avoid guessing in those cases.

  • Display content should degrade to a safe baseline rather than invent values.
  • Critical actions remain under user control, not hidden automation.
  • Validation is expected in solo, test, or controlled sessions first.
  • Trust is earned through conservative behavior, clear controls, and explicit scope.

Where to go next

Support and troubleshooting first steps

For immediate usage checks, use the troubleshooting guidance in the manual. Dedicated alpha support intake pages are being prepared; until then, use the alpha-path channels described on the VICA product page.
