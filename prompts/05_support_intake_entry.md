# Prompt: P1 support intake entry page

Create a static support intake entry page and placeholder template links.

## Read first

- `AGENTS.md`
- `CONTENT_PRINCIPLES.md`
- `WEBSITE_ROADMAP.md`

## Objective

Add a public support intake entry point that helps users send useful context without storing real customer data in-repo.

## Scope

- Static page only.
- Provide template links/placeholders (Markdown or HTML snippets).
- No backend submission endpoint, no account system, no CRM/webhook binding.

## Required output

- Clear instructions for what users should include (environment, issue type, steps tried).
- Prominent links to troubleshooting and trust/safety pages.
- Link/placeholder to downloadable template files.

## Validation and completion criteria

- Run `git status --short`.
- Run `git diff --check`.
- Commit locally; do not push.
